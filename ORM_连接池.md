# 连接池是什么？什么是ORM框架

client->web服务器->DB

应用服务器与数据库交互时，

1. 建立tcp连接
2. mysql认证
3. mysql执行并返回结果
4. mysql关闭
5. tcp关闭

新建tcp连接，响应时间长，导致QPS降低

## 数据库连接池：

数据库连接池：池子中放着数据库连接，需要就去池子中拿，用完归还。由他负责分配，管理，释放数据库连接池，它允许应用服务重复使用数据库连接，而不是重新建立

client->web应用->pool->db

1. 建立tcp连接
2. mysql认证
3. **mysql执行并返回结果 ** 只有这个是变换的，其他步骤是复用的。
4. mysql关闭
5. tcp关闭
